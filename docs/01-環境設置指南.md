# 01 - 環境設置指南 | Environment Setup Guide

## 前置條件 | Prerequisites

| 軟體 Software | 最低版本 Min Version | 說明 Note |
|--------------|---------------------|-----------|
| Docker | 24.x | 容器引擎 |
| Docker Compose | 2.x | 服務編排 |
| Python (選用) | 3.11+ | 本地測試用 |

## 首次設定步驟 | First-time Setup

### Step 1: 建立 `.env` 檔案

```bash
cp .env.example .env
```

然後用文字編輯器修改以下欄位：

| 設定 Setting | 說明 | 預設值 |
|-------------|------|--------|
| `JWT_SECRET_KEY` | JWT 簽名金鑰 (正式環境請改為隨機字串) | `dev-jwt-secret...` |
| `GPU_MOCK_MODE` | `true`=模擬, `false`=真實 GPU | `true` |
| `GPU_SERVER_1_HOST` | GPU 伺服器 1 IP | `192.168.1.100` |
| `GPU_SERVER_2_HOST` | GPU 伺服器 2 IP | `192.168.1.101` |
| `ANTHROPIC_API_KEY` | Claude API Key | `placeholder` |
| `OPENAI_API_KEY` | ChatGPT API Key | `placeholder` |
| `GOOGLE_API_KEY` | Gemini API Key | `placeholder` |

### Step 2: 啟動服務

```bash
# ZH: 建構並啟動所有容器 (包含核心與 AI 模塊) | EN: Build and start all containers
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml up -d --build

# ZH: 查看啟動日誌 | EN: View startup logs
docker-compose logs -f job-scheduler
```

### Step 3: 驗證服務

```bash
# ZH: 健康檢查 | EN: Health check
curl http://localhost:8002/health

# ZH: 預期回應 | EN: Expected response
# {"status":"healthy","service":"job-scheduler","version":"1.0.0",...}
```

### Step 4: 存取服務介面 | Access Services

當服務啟動後，您可以透過以下路徑存取平台各項功能：

| 服務項目 Service | 入口路徑 URL | 說明 Description |
|-----------------|-------------|-------------------|
| **Open WebUI** | `http://localhost/` | **AI 聊天介面**：主要對話式 AI 工具入口 (預設首頁)。 |
| **TRAIN_HUD** | `http://localhost/train/` | **訓練管理面板**：檢視 GPU 任務進度、配額與排程 (方案 C)。 |
| **API 文件** | `http://localhost:8002/docs` | **Swagger UI**：開發者測試用的 API 互動式手冊。 |
| **基礎監控** | `http://localhost/health` | **Nginx 狀態**：快速確認閘道器是否運作中。 |

## 🛠️ 開發與撰寫指引 | Development & Writing Guide

如果您需要根據需求修改介面、邏輯或增加訓練內容，請參考以下對應位置：

### 1. 修改網頁介面 (Frontend)
如果您要更換風格、修改按鈕文字或翻譯：
- **位置**：`CodeSpace/web-ui/`
- **主要檔案**：
    - `index.html`: 結構與靜態內容。
    - `styles.css`: 視覺風格（包含星空背景、亮暗模式變數）。
    - `app.js`: 交互邏輯與 **i18n 多語系字典目錄**。

### 2. 管理 GPU 資源與排程政策 (Resources)
如果您要增加新的 GPU 伺服器或是修改排程規則：
- **位置**：`CodeSpace/job-scheduler/app/`
- **關鍵檔案**：
    - `scheduler_policy.yaml`: **[最重要]** 伺服器 IP、硬體資訊、併發數限制都在此修改。

### 3. 編寫訓練腳本與後端邏輯 (Backend)
- **訓練腳本範例**：您可以建立訓練檔案於資源層目標位置，或透過 Job Scheduler API 進行指派。
- **後端 API 邏輯**：`CodeSpace/job-scheduler/app/`
    - `main.py`: 路由入口。
    - `scheduler.py`: 背景自動排程的核心邏輯。

## 服務管理 | Service Management

```bash
# ZH: 停止服務 | EN: Stop services
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml down

# ZH: 重新建構 (程式碼有更新時) | EN: Rebuild (after code changes)
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml up -d --build

# ZH: 查看所有容器狀態 | EN: Check all container status
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml ps

# ZH: 查看特定服務日誌 | EN: View specific service logs
docker-compose logs -f job-scheduler
docker-compose logs -f open-webui
docker-compose logs -f portkey
```

## 常見問題 | FAQ

### Q: 容器啟動失敗，顯示 port 已被佔用
```bash
# ZH: 查看佔用 port 的程式 | EN: Check which process uses the port
# Windows:
netstat -aon | findstr :8002
# Linux:
lsof -i :8002
```

### Q: SQLite 資料庫被鎖定
```bash
# ZH: 重啟 job-scheduler 容器 | EN: Restart job-scheduler
docker-compose restart job-scheduler
```

### Q: 需要重置資料庫
```bash
# ZH: 刪除資料庫檔案並重啟 | EN: Delete DB file and restart
rm data/ai_platform.db
docker-compose restart job-scheduler
```
