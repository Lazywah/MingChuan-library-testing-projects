## 模組化架構說明 | Modular Architecture

本系統採用多個 Docker Compose 檔案進行管理，您可以根據需求決定啟動的範圍：

| 檔案名稱 Filename | 包含服務 Services | 用途說明 Purpose |
| :--- | :--- | :--- |

## 快速管理指令 | Control Commands

### 🚀 全域控制 (啟動/關閉全部功能)
當您需要完整的 AI 訓練助理環境時，請使用組合指令。這對於確保模型釋放 GPU 資源非常重要。

```bash
# 一鍵啟動 (包含所有核心與模型)
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml up -d --build

# 一鍵關閉 (徹底清理所有服務與網路)
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml down
```

### 🛠️ 分層維運 (推薦開發使用)
如果您只是修改 UI 或業務邏輯，不需要頻繁重啟龐大的 AI 模型：

```bash
# 僅啟動/更新核心服務 (省去檢查模型容器)
docker-compose up -d

# 僅重啟後端排程器
docker-compose restart job-scheduler
```

## 🛠️ 開發與撰寫指引 | Development & Writing Guide

### 1. 修改網頁介面 (Frontend)
如果您要更換視覺風格、修改聊天室佈局：
- **位置**：`web-ui/`
- **關鍵檔案**：
    - `index.html`: 包含 Dashboard 與 Assistant 的分頁結構。
    - `app.js`: 處理串流對話 (Fetch Stream API) 與分頁切換。
    - `styles.css`: Cyberpunk 風格面板與對話氣泡樣式。

### 2. 管理 GPU 資源與排程政策 (Resources)
- **位置**：`job-scheduler/app/scheduler_policy.yaml`
- **說明**：在此定義實體 GPU 伺服器的節點池資訊。

### 3. 編寫 AI 助手與後端邏輯 (Backend)
- **位置**：`job-scheduler/app/routers/`
- **檔案**：
    - `chat.py`: **[新]** 負責 LLM 串流代理與紀錄存儲。
    - `jobs.py`: 負責訓練任務的 CRUD 與狀態更新。

## 服務管理 | Service Management

```bash
# 查看所有服務狀態
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml ps

# 重啟核心後端
docker-compose restart job-scheduler

# 停止服務 (保持容器)
docker-compose stop

# 關閉並移除容器 (常用)
docker-compose down

# 完全清理 (移除容器、網路與磁碟卷)
# 警告：這會刪除儲存在 Docker 卷中的資料
docker-compose down -v
```

## 🌐 遠端存取與部署 | Remote Access

本平台支援本機部署後供同區網內的其他機器存取。

### 1. 取得主機 IP
在執行 Docker 的主機上執行：
```powershell
ipconfig   # Windows
# 尋找 IPv4 位址 (例如: 192.168.1.105)
```

### 2. 跨機器連結
在另一台電腦的瀏覽器中輸入：
- `http://[主機IP]/train/`

### 3. 防火牆設定 (重要)
若其他機器無法連線，請檢查主機的防火牆設定：
- **開放埠號**：確保 **Port 80** (HTTP) 與 **Port 8002** (API Docs) 已設為防火牆允許規則 (Inbound Rules)。
- **網路類型**：建議將網路設為「專用 (Private)」。

## 常見問題 | FAQ

### Q: 為什麼 AI 助手沒有回覆？
1. 請確認 `ai-platform-portkey` 容器是否正常運作。
2. 檢查 `.env` 中的 API Key 是否正確設定。
3. 查看日誌：`docker-compose logs -f job-scheduler` 確認串流轉發有無錯誤。

### Q: 如何切換回舊版 Open WebUI？
您可以存取 `http://localhost/` 直接使用 Open WebUI，但該處對話紀錄與 `TRAIN_HUD` 物理隔離。
