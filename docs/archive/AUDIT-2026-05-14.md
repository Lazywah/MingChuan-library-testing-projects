### C-3｜`/usage/increment` 對所有使用者開放且超額後仍扣減
- **檔案**：`job-scheduler/app/routers/auth.py` 行 304–323
- **問題**（兩個子問題）：
  1. **開放給一般使用者**：端點使用 `current_user: models.User = Depends(get_current_user)`，
     任何登入使用者（學生）均可呼叫此 API 自行增加 Token 計數。
     API 的設計意圖是「供 Job Scheduler 或 Portkey 呼叫」，但並未限制角色。
  2. **超額後仍扣減**：呼叫 `crud.increment_token_usage()`（先加再檢查）後，才用 `>`
     檢查是否超額並回傳 HTTP 429——但此時 Token 已經被加上去了。
     正確做法應先檢查、再扣減（或用原子 UPDATE）。
- **修法**：
  - 限制呼叫角色：`_: models.User = Depends(require_role("admin"))` 或改為內部呼叫，不暴露為公開 API。
  - 改用 `crud.try_deduct_tokens()` 取代 `crud.increment_token_usage()`。

### C-4｜自助註冊可設定 `role=admin`
- **檔案**：`job-scheduler/app/schemas.py` 行 40
- **問題**：`UserCreate.role: Optional[str] = "student"` 未設驗證約束。
  任何人可在 POST `/register` 請求中傳入 `"role": "admin"`，直接建立管理員帳號。
- **修法**：加入 pydantic `@field_validator`：
  ```python
  @field_validator("role")
  @classmethod
  def role_must_be_student(cls, v):
      # 公開註冊只允許 student；teacher/admin 由管理員配發
      if v not in ("student",):
          raise ValueError("Public registration only allows role=student")
      return v
  ```

### C-5｜`datasets.py` 路徑穿越漏洞 (Path Traversal)
- **檔案**：`job-scheduler/app/routers/datasets.py` 行 28–31
- **問題**：`safe_filename = f"{uuid4_prefix}_{file.filename}"` 後直接
  `os.path.join(user_dataset_dir, safe_filename)` 組成路徑。
  若 `file.filename = "../../etc/cron.daily/evil"` 則路徑變為
  `/data/datasets/{user_id}/abc12345_../../etc/cron.daily/evil`，
  `os.path.join` 不會阻止 `../` 跳出目錄。
- **修法**：
  ```python
  import pathlib
  safe_name = pathlib.Path(file.filename).name  # 只取最後檔名元件
  safe_filename = f"{uuid4_prefix}_{safe_name}"
  # 再用 os.path.realpath 確認結果在允許目錄內
  resolved = os.path.realpath(os.path.join(user_dataset_dir, safe_filename))
  if not resolved.startswith(os.path.realpath(user_dataset_dir)):
      raise HTTPException(400, "Invalid filename")
  ```

### C-8｜前端 XSS — `createBubble` 歷史訊息注入

### C-9｜前端 SSE 串流未攜帶 JWT Token → 認證永遠失敗
- **檔案**：`web-ui/app.js` 行 1581
- **問題**：
  ```javascript
  currentEventSource = new EventSource(`${API_BASE}/jobs/${jobId}/stream`);
  ```
  `EventSource` 是瀏覽器原生 API，**不支援自訂 headers**，因此不會傳送
  `Authorization: Bearer <token>`。後端 `stream_job_logs` 需要 `Depends(get_current_user)`，
  所有 SSE 請求都會收到 HTTP 401，日誌功能根本無法運作。
- **修法**（擇一）：
  - **Option A**：改用 `fetch` + `ReadableStream` 來取代 `EventSource`，
    可自訂 headers：
    ```javascript
    const resp = await fetch(`${API_BASE}/jobs/${jobId}/stream`, {
        headers: { 'Authorization': `Bearer ${authToken}` }
    });
    ```
  - **Option B**：後端改用短期 one-time token（query param），
    `GET /jobs/{id}/stream?stream_token=<short_lived_jwt>`。

### C-10｜前端 `fetchJobs` 取錯欄位 → 任務列表永遠為空
- **檔案**：`web-ui/app.js` 行 1069
- **問題**：
  ```javascript
  const jobs = data.items || data || [];
  ```
  後端 `GET /jobs` 回傳 `{total: N, jobs: [...]}` (schema: `JobListResponse`)。
  `data.items` 不存在（undefined/falsy），`data` 是整個物件（truthy），
  所以 `jobs = data`（整個物件）。之後 `jobs.filter(...)` 呼叫物件的不存在方法，丟出 TypeError，
  或被 catch 吃掉後顯示空狀態。
- **修法**：
  ```javascript
  const jobs = data.jobs || [];
  ```

### H-5｜`datasets.py` 無個人儲存空間限制
- **檔案**：`job-scheduler/app/routers/datasets.py` 行 14–81
- **問題**：每位使用者可上傳任意數量/大小的檔案。nginx 限制 10GB 單檔，
  但多次上傳可填滿整個磁碟分區。
- **修法**：在上傳前計算使用者目錄總大小，超過閾值拒絕（如 2GB/人）：
  ```python

  if used + file.size > MAX_USER_STORAGE:
      raise HTTPException(413, "Storage quota exceeded")
  ```

### H-6｜`worker.py (take_job)` 搶佔失敗後不嘗試下一筆任務
- **檔案**：`job-scheduler/app/routers/worker.py` 行 93–117
- **問題**：Worker 呼叫 `get_pending_jobs()` 取得全部待處理任務，選 `pending_jobs[0]`，
  但若原子 UPDATE 失敗（已被另一節點搶佔，`rowcount=0`），直接回傳 `{"job": None}`。
  這意味著若優先級最高的任務剛好被搶走，其他仍在等待的任務也無法被分配，
  必須等下一個輪詢週期（預設 5 秒）。
- **修法**：改為迴圈嘗試清單中的下一筆任務，直到成功搶佔或清單用盡：
  ```python
  for job in pending_jobs:
      result = db.execute(update(...).where(...).values(...))
      db.commit()
      if result.rowcount > 0:
          db.refresh(job)
          return {"job": {...}}
  return {"job": None}
  ```

### M-1｜`database.py` 手動 migration 靜默吞錯
- **檔案**：`job-scheduler/app/database.py` 行 136–167
- **問題**：所有 `ALTER TABLE` 以 `try/except Exception: pass` 包裹，

  也沒有追蹤哪些 migration 已執行，重啟時每次都執行全部 ALTER。
- **修法**：改用 Alembic（已在 `requirements.txt` 中，但未使用），
  或至少只捕捉 `OperationalError` 中「duplicate column」的情況。

### M-2｜`TrainingJob` 缺少 `updated_at` 欄位

### M-3｜`crud.append_job_log` 每次都 `db.commit()`
- **檔案**：`job-scheduler/app/crud.py` 行 357–365
- **問題**：GPU Worker 每行日誌都呼叫 `report_update` → `append_job_log` → `db.commit()`。
  對於高頻輸出的訓練腳本（如每 batch 輸出一行），每秒可能觸發數十次寫入。
  SQLite 在 WAL 模式下尚可接受，但仍有明顯效能損耗。
- **修法**：緩衝日誌，改為批次 commit（如每 10 條或每 5 秒）；
  或後端接受 `{"log": "..."}` 且每 N 行才寫一次 DB。

### M-4｜`email_service.py` 使用同步 `smtplib`，可能阻塞 Worker Thread
- **檔案**：`job-scheduler/app/services/email_service.py` 行 23–42
- **問題**：`smtplib.SMTP` 是同步 I/O。雖然透過 `BackgroundTasks` 執行，

- **問題**：`send_login_alert` 在每次登入（包含頻繁的自動重新整理觸發的重新登入）都寄信，
  若 SMTP 啟用，使用者信箱會被反覆填滿。
- **修法**：設定條件：「只在新 IP 登入」或「超過 24 小時未登入」時才寄警告信。

### L-4｜`nginx.conf` 缺少安全性 HTTP Headers
- **檔案**：`infrastructure/nginx.conf`
- **問題**：缺少 `X-Frame-Options`、`X-Content-Type-Options`、
  `Content-Security-Policy`、`Referrer-Policy` 等現代安全標頭。
- **修法**（加在兩個 `server` block 內）：
  ```nginx
  add_header X-Frame-Options "SAMEORIGIN" always;
  add_header X-Content-Type-Options "nosniff" always;
  add_header Referrer-Policy "strict-origin-when-cross-origin" always;
  ```

### L-5｜`app.js` 診斷用 `console.log` 應在正式版移除
- **檔案**：`web-ui/app.js` 行 1056, 1063, 1069, 1004–1010
- **問題**：多處 `console.log('Jobs data received:', data)` 等偵錯輸出，
  會在瀏覽器 DevTools 中洩漏內部資料結構。
- **修法**：正式版打包時移除，或改用環境判斷：
  ```javascript
  if (window.__DEV__) console.log(...)
  ```

### L-6｜`admin.py` 刪除使用者後未清理 `ChatHistory`
- **檔案**：`job-scheduler/app/routers/admin.py` 行 206
- **問題**：刪除使用者前只清了 `TokenUsage`，忘記清 `ChatHistory`：
  ```python
  db.query(models.TokenUsage).filter(...).delete()
  db.delete(db_user)
  ```
  `ChatHistory` 有 `ForeignKey("users.id", ondelete="CASCADE")`，
  故 SQLite 啟用 FK 時會自動刪除，但如果 FK 未啟用（某些情境）則會殘留孤兒資料。
- **說明**：`database.py` 已設定 `PRAGMA foreign_keys=ON`，因此 CASCADE 會正常觸發。
  此項目是低優先的防禦性改進：加入顯式刪除確保可移植性。
