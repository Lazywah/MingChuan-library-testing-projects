## 🌍 i18n 雙語功能開發 | Internationalization

前端採用自研的輕量級 i18n 引擎（位於 `app.js` 的 `applyLanguage`），開發時請遵循：

### 1. 新增語系字串
在 `app.js` 的 `TRANSLATIONS` 物件中，分別在 `zh` 與 `en` 項目下新增相同名稱的 Key。

### 2. 標記 HTML 標註
- **內容文字**：使用 `data-i18n="YOUR_KEY"`。
- **輸入框佔位符**：使用 `data-i18n-placeholder="YOUR_KEY"`。
- **無障礙標籤 (Aria)**：使用 `data-i18n-aria="YOUR_KEY"`。

> [!TIP]
> **重要規則**：如果標籤內含圖示 (Ion-Icons)，請將 `data-i18n` 置於內層的 `<span>` 上，以避免圖示被翻譯文字覆蓋。

## 如何新增功能 | How to Add Features

```
Module 1 (config)
    ↓
Module 2 (database)
    ↓
Module 3 (models)    Module 4 (schemas)    Module 7 (gpu_client)
    ↓                    ↓                      ↓
Module 5 (crud)  ←───────┘                 Module 8 (scheduler)
    ↓
Module 6 (auth)
    ↓
routers/auth.py    routers/jobs.py
    ↓                    ↓
Module 9 (main.py) ← 組裝所有模組
```

## 如何新增功能 | How to Add Features

### 新增 API 端點

1. 在 `schemas.py` 加入請求/回應模型
2. 在 `crud.py` 加入資料庫操作
3. 在 `routers/` 中建立或修改路由
4. 在 `main.py` 中 `include_router`

### 新增資料庫表

1. 在 `models.py` 加入新的 ORM Class
2. 在 `infrastructure/schema.sql` 加入對應 SQL
3. 重啟服務 (自動 CREATE IF NOT EXISTS)

### 新增 GPU 後端 (如 Slurm)

1. 在 `gpu_client.py` 繼承 `BaseGPUClient`
2. 實作 `connect()`, `get_available_gpus()`, `execute_training()` 等方法
3. 在 `get_gpu_client()` 工廠函式中加入新條件

### 新增 LLM Provider

1. 在 `portkey/config.yaml` 加入新 Provider
2. 在 `.env` 加入對應 API Key
3. 重啟 Portkey 容器

## 本地開發 (不使用 Docker) | Local Development

```bash
cd job-scheduler
python -m venv .venv
source .venv/bin/activate   # Linux/Mac
# .venv\Scripts\activate    # Windows (工作站本地開發時使用)
pip install -r requirements.txt

# 設定環境變數
export DATABASE_PATH=../data/ai_platform.db   # Linux/Mac
export GPU_MOCK_MODE=true
# set DATABASE_PATH=../data/ai_platform.db    # Windows (工作站本地開發時使用)
# set GPU_MOCK_MODE=true

# 啟動
uvicorn app.main:app --reload --port 8002
```
