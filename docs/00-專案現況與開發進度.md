## 3. API 端點總覽 (API Endpoint Map)

### 認證模組 `/api/v1/auth/`
| 方法 | 路徑 | 說明 | 狀態 |
|------|------|------|------|
| POST | `/login` | 表單登入 (username + password) | ✅ |
| POST | `/register` | 使用者自助註冊 | ✅ |
| GET  | `/me` | 取得當前使用者資訊 + Token 用量 | ✅ |
| PUT  | `/me/update` | 更新個人資料 (email / password / tutorial) | ✅ |
| GET  | `/me/token-usage` | 查詢 Token 用量詳細 | ✅ |

### SSO 模組 `/api/v1/auth/`
| 方法 | 路徑 | 說明 | 狀態 |
|------|------|------|------|
| POST | `/sso/login` | Mock SSO 登入 (查 sso_policy.yaml) | ✅ |
| GET  | `/sso/cas/login` | 真實 CAS 協定入口 | ❌ 未實作 |

### 訓練任務模組 `/api/v1/jobs/`
| 方法 | 路徑 | 說明 | 狀態 |
|------|------|------|------|
| POST | `/` | 提交新任務 (扣 Token) | ✅ |
| GET  | `/` | 列出任務 (RBAC: student=自己, teacher/admin=全部) | ✅ |
| GET  | `/{job_id}` | 查詢單一任務狀態 | ✅ |
| DELETE | `/{job_id}` | 取消任務 (pending/queued 限定) | ✅ |
| GET  | `/{job_id}/stream` | SSE 串流日誌 + 指標 | ✅ |

### 聊天模組 `/api/v1/chat/`
| 方法 | 路徑 | 說明 | 狀態 |
|------|------|------|------|
| POST | `/completions` | SSE 串流聊天 | ⚠️ Mock Echo |
| GET  | `/history` | 取得對話歷史 | ✅ (session_id 固定) |

### GPU Worker 模組 `/api/v1/worker/`
| 方法 | 路徑 | 說明 | 狀態 |
|------|------|------|------|

## 6. 建議優先事項 (Next Steps)

### 立即可做：啟動 Portkey 並設定 API 金鑰
聊天功能已接通 Portkey 代理，但需要實際的 AI 服務才能使用：
```bash
# 1. 啟動 AI Models 服務（含 Portkey）
docker compose -f docker-compose.ai-models.yml up -d

# 2. 在 .env 加入 API 金鑰（擇一）
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
GOOGLE_API_KEY=...
# 或使用 Ollama 本機模型（不需要 API 金鑰）
```

### ✅ 已完成：GPU Worker 心跳端點 (2026-05)
Worker 端已實作時間驅動心跳（`HEARTBEAT_INTERVAL`，預設 30s），服務層與管理介面叢集儀表板均已對接，無需再做任何操作。

### 第一步：消除 `batch_update_tokens` N+1 (2 小時)
`admin.py` 的批量 Token 操作目前逐筆查詢，改為 SQLAlchemy bulk UPDATE：
```python
db.query(models.TokenUsage).filter(
    models.TokenUsage.user_id.in_(payload.user_ids)
).update({models.TokenUsage.tokens_limit: payload.value}, synchronize_session=False)
```

### 第三步：導入 Alembic 資料庫 Migration (1 天)
目前開發階段以 `Base.metadata.create_all()` 自動建表，正式上線前建議：
```bash
pip install alembic
alembic init migrations
# 將現有 models.py 作為初始版本
alembic revision --autogenerate -m "initial schema"
```

### 第三步：AI Hub 功能擴展 (依需求排序)
建議優先開發：
1. **知識庫/RAG** — 與圖書館情境最相關，需 vector DB（如 Chroma）

3. **多語翻譯** — 可直接用聊天 API + system prompt 實現，無需額外後端

### 第四步：Notebook 進階功能 (選擇性)

4. **自訂環境變數格子** — 讓使用者在 UI 填寫 API Key 等，自動注入 Docker 容器 env
