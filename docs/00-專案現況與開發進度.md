## 5. 差距分析 (Gap Analysis)

### ✅ 已解決 (2025-05 開發衝刺)

| 問題 | 解決方式 |
|------|----------|
| 聊天 Mock Echo 模式 | `chat.py` 改為 httpx 串流 proxy 到 Portkey |
| 模型選擇被忽略 | `model_id` 傳入 Portkey payload + provider 自動路由 |
| `session_id` 寫死 | 從 `ChatRequest` 讀取 / 生成 UUID |
| `scheduler.py` 閒置 | 加入每 5 分鐘超時清理迴圈 |
| 無速率限制 | slowapi `@limiter.limit("10/minute")` on login |
| `datetime.utcnow()` 棄用警告 | 全面改為 `datetime.now(timezone.utc)` |
| Worker race condition | 原子 SQL UPDATE + rowcount 檢查 |
| Worker token 比較安全漏洞 | `hmac.compare_digest` 防時序攻擊 |
| 列表回應含 logs 欄位 | 新增 `JobListItem` schema，排除 logs |
| 無測試覆蓋 | 50 個測試 (22 CRUD + 28 API)，全數通過 |
| Nginx 啟動失敗 (open-webui) | 使用延遲 DNS 解析，open-webui 不啟動時仍可運行 |

### 🟡 中優先 — 影響完整性

| 問題 | 影響 | 說明 |
|------|------|------|
| **11 個 AI Hub 功能為 Coming Soon** | 平台功能顯著不完整 | 各功能需要不同後端支援 (圖片生成 API、RAG 知識庫等) |
| **叢集 GPU 統計為空** | Admin 看不到實際 GPU 使用狀況 | `/admin/cluster/stats` 直接返回 `[]`；需 Worker 定時上報心跳 |
| **無 SQLAlchemy 外鍵約束** | DB 可能產生孤立資料 | `user_id` 等欄位只有邏輯關聯，刪除 user 時若沒手動清理可能留下殘留 |

### 🟢 低優先 — 技術債

| 問題 | 影響 | 說明 |
|------|------|------|
| **真實 CAS SSO 未實作** | 只能用 Mock 模式 | `sso_client.py` 中有框架但無 CAS ticket 驗證流程 |
| **app.js 超過千行** | 前端維護難度上升 | 所有業務邏輯集中於單一檔案，未來建議拆分為 ES6 模組 |
| **Pydantic v2 class Config 棄用警告** | 未來 v3 移除 | schemas.py / config.py 改用 `model_config = ConfigDict(...)` |

### 立即可做：啟動 Portkey 並設定 API 金鑰
聊天功能已接通 Portkey 代理，但需要實際的 AI 服務才能使用：
```bash
# 1. 啟動 AI Models 服務（含 Portkey）
docker compose -f docker-compose.ai-models.yml up -d

# 2. 在 .env 加入 API 金鑰（擇一）
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
GOOGLE_API_KEY=...
# 或使用 Ollama 本機模型（不需要 API 金鑰）
```

### 第一步：Worker 心跳上報 (半天)
讓 `worker.py` 在 poll loop 中定期 POST 心跳到新端點 `/api/v1/worker/heartbeat`，服務層儲存最後心跳時間與 GPU 使用率，讓 `/admin/cluster/stats` 有資料可回傳。

### 第二步：加入 SQLAlchemy 外鍵 (1 天)
在 `models.py` 的 `user_id` 欄位加入 `ForeignKey("users.id")`，並確保 `crud.py` 的刪除使用者邏輯（已存在）覆蓋所有關聯表。

### 第三步：AI Hub 功能擴展 (依需求排序)
建議優先開發：
1. **知識庫/RAG** — 與圖書館情境最相關，需 vector DB (如 Chroma)
2. **圖片辨識** — 僅需將現有 chat proxy 改為 vision model
3. **多語翻譯** — 可直接用聊天 API + system prompt 實現
