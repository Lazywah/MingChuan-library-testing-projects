# 06 - 專案架構與檔案說明 | Project Structure & File Guide

本文件提供了 AI 訓練平台 (`CodeSpace`) 根目錄下所有重要資料夾與檔案的結構概覽與簡短說明，幫助開發者與維護者快速掌握專案全貌。

## 📁 根目錄結構 (CodeSpace)

| 檔案 / 資料夾 | 說明 |
| --- | --- |
| `.env` / `.env.example` | 專案的環境變數設定檔，包含金鑰、資料庫路徑、GPU 連線資訊等。 |
| `.gitignore` | Git 版本控制忽略清單，確保不提交環境設定或本地資料庫檔案。 |
| `docker-compose.yml` | Docker 服務編排設定檔，用於一鍵啟動平台的所有微服務 (Nginx, API, WebUI 等)。 |
| `README.md` | 專案的主介紹文件、快速啟動指南及架構總覽。 |
| `data/` | 容器資料持久化的本機掛載目錄（如 SQLite 的 `ai_platform.db` 檔案會存在這）。 |
| `docs/` | 專案的開發、部署、API 與架構等相關說明文檔目錄。 |
| `infrastructure/` | 基礎設施設定檔，如 Nginx 的反向代理設定、資料庫的初始建表 SQL。 |
| `job-scheduler/` | **核心微服務**，負責帳號認證與 GPU 訓練任務的排程與分發 (使用 FastAPI 開發)。 |
| `portkey/` | Portkey (LLM Gateway) 網關微服務的設定檔。 |
| `scripts/` | 部署或系統維護用的輔助執行指令碼 (如 `deploy.sh`)。 |
| `tests/` | 自動化測試腳本目錄。 |
| `web-ui/` | **核心入口介面**：深度整合的訓練任務管理與 AI 助手前端介面存放處。 |

## 📁 `job-scheduler` 模組內部結構

核心後端系統採用「積木式模組化」設計，每一個 Python 檔案都有專責的任務：

```text
job-scheduler/
├── Dockerfile                  # 該服務的容器建構描述檔
├── requirements.txt            # Python 依賴套件清單
└── app/
    ├── __init__.py
    ├── main.py                 # (Module 9) FastAPI 應用主入口、組裝所有路由與模組
    ├── config.py               # (Module 1) 讀取並檢驗 .env 的 Pydantic 設定檔
    ├── database.py             # (Module 2) SQLite 連線引擎與 WAL 模式設定
    ├── models.py               # (Module 3) SQLAlchemy ORM 資料表模型設計
    ├── schemas.py              # (Module 4) Pydantic API 請求與回應的資料驗證模型
    ├── crud.py                 # (Module 5) 負責所有對資料庫的 CRUD（增刪改查）操作
    ├── auth.py                 # (Module 6) JWT Token 發行與驗證、RBAC 權限裝飾器
    ├── gpu_client.py           # (Module 7) GPU 通訊類別 (支援 Mock 模擬與 SSH 實體連線)
    ├── scheduler.py            # (Module 8) 背景非同步排程器，監控佇列並分發工作給 GPU
    └── routers/                # API 路由層
        ├── __init__.py
        ├── auth.py             # 登入、註冊、Token 查詢等 API 端點
        ├── jobs.py             # 訓練任務管理 API 端點
        └── chat.py             # **[新]** AI 助手串流代理與歷史紀錄 API
```

## 📁 `infrastructure` 基礎設施結構

| 檔案 | 說明 |
| --- | --- |
| `nginx.conf` | Nginx 反向代理配置，負責將 80 port 的請求依據路徑分發給 WebUI 或是 API 服務。 |
| `schema.sql` | 定義最初的 SQLite 資料表結構與索引 (Users, Models, Servers, Jobs 等)。 |
