## 👥 角色權限說明 | Role Overview

系統共有三種角色，帳號由管理員配發：

| 角色 | 說明 | 訓練任務 | AI 助手 | Admin Panel |
|------|------|:---:|:---:|:---:|
| **student（學生）** | 一般使用者，僅能查看自己的任務 | 僅自己 | ✅ | ❌ |
| **teacher（教師）** | 可監控全系統任務，適合課堂管理 | 所有人 | ✅ | ❌ |
| **admin（管理員）** | 完整系統控制，含帳號與模型管理 | 所有人 | ✅ | ✅ |

> [!NOTE]
> Token 月度配額由管理員統一設定，各角色皆受配額限制。教師帳號的配額預設與學生相同，管理員可依需求個別調整。

## 🚀 快速上手 | Getting Started

在進入系統後，您會先看到 **教學導覽**，之後直接進入 **運算任務 (Compute Tasks)** 頁面。

### 1. 登入系統
- 請使用提供的帳號與密碼進行登入。
- SSO 模擬使用者可直接以學號同時作為帳號與密碼登入 (如 `T1090001` / `T1090001`)。
- 系統採用的 **JWT 安全授權** 具備自動到期機制，若登入失效請重新刷新頁面。

### 2. 入站教學
- 首次登入時，系統會自動顯示 **「Welcome to AI Base!」** 的教學面板。
- 導覽介紹三大板塊：**運算任務**、**AI 助手**、**系統設定**。
- 勾選「不再顯示」可永久關閉；前往「設定 → 教學手冊 → 開啟教學」可重新開啟。

## ⚡ 運算任務操作 | Compute Tasks Guide

運算任務頁面是您提交與監控訓練任務的核心位置（原「儀表板」已更名為此）。頁面上方有三個子頁籤：

---

### 🚀 Notebook 頁籤（v2.0 Lab — VS Code in Browser）

完整的 VS Code 環境在瀏覽器內運作，含終端機、檔案總管、Notebook 編輯與 Run on GPU。
（取代 v1 偽 Notebook，使用 code-server + per-user persistent volume。）

#### 啟動 Notebook

1. 進入 **運算任務 → Notebook** 頁籤
2. 從下拉選單選擇 **Image**（PyTorch / TensorFlow / HuggingFace / llama.cpp / vLLM / Dev Tools）
3. 點擊 **「開啟 Notebook」** → 5–10 秒內跳轉至 `/code/{user_id}/` 開啟 VS Code 分頁

#### 在 VS Code 內可做的事

| 功能 | 操作 |
|------|------|
| **撰寫程式碼** | 任意 `.py`、`.ipynb`、`.cpp`、`.cu`、`.sh` 檔案，內建語法高亮與自動補全 |
| **多 Terminal** | 點 ⌃` 或左下「Terminal」面板，開啟 bash session（`ls`、`pip install --user xxx`、`tail -f`、`vim`、`htop` 等）|
| **檔案總管** | 左側 Explorer 顯示 `/home/coder/`，可拖曳上傳 / 下載 |
| **Notebook (.ipynb)** | 內建 Jupyter extension，逐格編輯與註解 |
| **Git** | base image 預裝 `git` 與 `gh` CLI |
| **AI Base: Run on GPU** | 右鍵 Python 檔或 Notebook cell → submit 為 GPU job，輸出串流至 VS Code「AI Base GPU」Output Panel |
| **AI Base: Pick GPU Node** | 從 Command Palette (⇧⌘P) 選擇特定 GPU 節點 |
| **AI Base: Pick Framework Image** | 切換訓練 Job 使用的 Docker image |

#### 環境持久化（三層架構）

| 層級 | 內容 | 跨次保留？ |
|------|------|----------|
| Base image | 預裝 PyTorch / transformers / accelerate / bitsandbytes / numpy / sklearn / git / conda | ✅ 學期內鎖定 |
| Per-user volume `/home/coder/` | `~/.local/`（pip --user）、`~/.cache/huggingface`、自己的 projects、checkpoints | ✅ idle 30 分後容器停，volume 永久保留 |
| Shared `/opt/models/` | 預下載熱門模型（Llama-2-7B、Mistral-7B、BERT、T5 等） | ✅ 唯讀掛載，所有使用者共用 |

#### Secrets 注入

需要 HF_TOKEN、WANDB_API_KEY 等 API key？

1. 進入 **系統設定 → Secrets 管理**
2. 新增 secret（name 須符合 ENV_VAR 格式，value 以 AES-256-GCM 加密儲存）
3. 之後使用「Run on GPU」時，系統會**自動把所有 secrets 注入到 GPU 容器**為環境變數

> ⚠️ value 一旦儲存就無法再讀取（管理員也不行），只能更新或刪除。

#### Idle 自動關閉

- 容器在 **idle 30 分鐘** 後自動停止（檔案保留於 volume）
- 再次點「開啟 Notebook」會重新啟動，所有檔案還原
- 每日 session 累積時長可在「狀態卡片」查看

## 🤖 AI 助手操作 | AI Assistant (Hub)

全新的 AI 大廳採用分類導覽機制。

### 1. 導覽大廳 (AI Hub)
- **AI 模型**：進行文字聊天、圖片分析與知識搜尋。
- **文書寫作**：輔助撰寫論文摘要、合約分析或會議簡報。
- **影音創作**：生成示意圖、腳本與背景音樂。

### 2. 對話功能
- **串流回傳**：AI 會隨想隨讀，實現即時反應。
- **多對話管理**：點擊左側「開始新對話」可建立獨立的討論串；點擊歷史紀錄可切換回舊對話。
- **帳號隔離**：每位使用者的聊天紀錄獨立儲存，不同帳號登入只會看到自己的對話。
- **返回大廳**：若需切換分類功能，請點擊對話視窗上方的「🔙 返回大廳」按鈕。

## ⚙️ 系統設定 | Settings

設定頁面包含以下功能區塊：

| 區塊 | 說明 |
|------|------|
| **Token Resources** | 代幣配額圓環進度條，顯示已使用量、總配額與重置日期。 |
| **Profile Update** | 個人資料變更，支援隨時修改電子信箱與密碼。採跨層驗證機制。 |
| **Appearance** | 深色/淺色主題切換。 |
| **Localization** | 中英文語言即時切換。 |
| **Tutorial Guide** | 「開啟教學」按鈕，可重新顯示入站教學面板。 |
| **Secrets 管理** *(v2.0)* | 儲存 API key（HF_TOKEN / WANDB_API_KEY 等），提交 GPU 任務時自動注入容器環境變數。所有 value AES-256-GCM 加密，**儲存後無法再讀取**（含管理員），只能更新或刪除。 |
| **Account** | 登出系統按鈕。 |

> [!NOTE]
> **佈局說明**：設定頁面已全面優化為「單欄式橫向條列佈局 (Responsive Horizontal Bars)」，確保在任何螢幕尺寸下都能精確操作各項選項。

## 📋 教師功能 | Teacher Features

以 **teacher** 角色登入時，介面與學生相同，但以下功能有所不同：

### 全域任務監控
- 在「運算任務 (Compute Tasks)」頁面，教師可看到**所有使用者**提交的訓練任務，而非僅限自己的任務。
- 任務卡片會額外顯示提交者帳號，方便課堂掌握學生訓練進度。
- 教師仍**無法取消**他人的任務（該操作僅限管理員）。

### 其他功能
- AI 助手、系統設定、Token 配額查詢等功能與學生完全相同。
- 若需進行帳號管理或系統設定，請聯絡管理員。

## 🛡️ 管理員功能 | Admin Panel

當以 **admin** 角色帳號登入時，導覽列會多出一個 **「管理中心 (Admin)」** 入口（網址：`/admin-ui/`）。

### 使用者管理
- 查看所有帳號的信箱、角色、在線狀態、最後登入 IP / 時間及 Token 用量。
- **建立帳號**：點擊「Provision User」配發新帳號，系統自動產生臨時密碼並寄信通知。
- **編輯帳號**：點擊列表中的使用者可修改角色、啟用狀態、Token 上限與密碼（需輸入管理員密碼解鎖）。
- **批量設定 Token 上限**：輸入新額度後點擊「Apply to All」套用至全體使用者。
- **批量歸零用量**：點擊「Reset All Usage」將所有使用者的本月已用量歸零。
- **個別歸零用量**：在使用者編輯視窗點擊「Reset Usage」，僅歸零該使用者的用量。

### 模型管理
- 新增、編輯、刪除 AI 模型。支援 **API 模型**（雲端：Anthropic / OpenAI / Google / Ollama）與**本地模型**兩種類型。

### 全域排程
- 查看全系統的訓練任務，可篩選狀態（pending / running / completed 等）。
- **強制取消**任務或**調整優先級**（0–5，數字越大越先執行）。

### 系統設定檔
- 直接在管理介面編輯 `.env`、`scheduler_policy.yaml`、`sso_policy.yaml` 等設定檔，儲存後即時生效。

### 數據分析
- 查看各學系 Token 用量分布圖與工具使用佔比，支援依學系篩選。

## ❓ 常見問題 | FAQ

### Q: 如果 AI 回到一半出現紅色提示？
這是由於網路波動或配額不足造成的中斷。系統會**自動保留**已生成的文字內容，您可以嘗試重新發送訊息或到設定頁面檢查 Token 餘額。

### Q: 如何切換中英文介面？
請前往「系統設定 (Settings)」分頁，點擊「語言切換」按鈕即可。大廳與對話介面將會即時同步變更。

### Q: 代幣配額在哪裡查看？
代幣配額已移至「系統設定 (Settings)」分頁的第一個區塊中。
