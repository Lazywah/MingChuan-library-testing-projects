# 05 - 開發者指南 | Developer Guide

## 模組架構圖 | Module Architecture

```
main.py (Module 9)
  │
  ├── config.py (Module 1)      ← 設定管理 (基礎)
  ├── database.py (Module 2)    ← SQLite 連線 (依賴 Module 1)
  ├── models.py (Module 3)      ← ORM 模型 (依賴 Module 2)
  ├── schemas.py (Module 4)     ← 資料驗證 (獨立)
  ├── crud.py (Module 5)        ← 資料操作 (依賴 Module 3, 4)
  ├── auth.py (Module 6)        ← JWT 認證 (依賴 Module 1, 5)
  ├── gpu_client.py (Module 7)  ← GPU 通訊 (獨立)
  ├── scheduler.py (Module 8)   ← 排程器 (依賴 Module 5, 7)
  │
  └── routers/
      ├── auth.py               ← 認證路由 (依賴 Module 5, 6)
      └── jobs.py               ← 任務路由 (依賴 Module 5, 6)
```

## 依賴關係 | Dependency Graph

```
Module 1 (config)
    ↓
Module 2 (database)
    ↓
Module 3 (models)    Module 4 (schemas)    Module 7 (gpu_client)
    ↓                    ↓                      ↓
Module 5 (crud)  ←───────┘                 Module 8 (scheduler)
    ↓
Module 6 (auth)
    ↓
routers/auth.py    routers/jobs.py
    ↓                    ↓
Module 9 (main.py) ← 組裝所有模組
```

## 如何新增功能 | How to Add Features

### 新增 API 端點

1. 在 `schemas.py` 加入請求/回應模型
2. 在 `crud.py` 加入資料庫操作
3. 在 `routers/` 中建立或修改路由
4. 在 `main.py` 中 `include_router`

### 新增資料庫表

1. 在 `models.py` 加入新的 ORM Class
2. 在 `infrastructure/schema.sql` 加入對應 SQL
3. 重啟服務 (自動 CREATE IF NOT EXISTS)

### 新增 GPU 後端 (如 Slurm)

1. 在 `gpu_client.py` 繼承 `BaseGPUClient`
2. 實作 `connect()`, `get_available_gpus()`, `execute_training()` 等方法
3. 在 `get_gpu_client()` 工廠函式中加入新條件

### 新增 LLM Provider

1. 在 `portkey/config.yaml` 加入新 Provider
2. 在 `.env` 加入對應 API Key
3. 重啟 Portkey 容器

## 本地開發 (不使用 Docker) | Local Development

```bash
cd job-scheduler
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt

# 設定環境變數
set DATABASE_PATH=../data/ai_platform.db   # Windows
set GPU_MOCK_MODE=true

# 啟動
uvicorn app.main:app --reload --port 8002
```
