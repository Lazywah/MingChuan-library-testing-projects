## 1. 前置檢查

### 硬體需求

| 角色 | CPU | RAM | 磁碟 | GPU |
|------|-----|-----|------|-----|
| **服務層** (你現在裝的這台) | 4 核+ | 8 GB+ | 50 GB+ SSD | 不需要 |
| **GPU 工作節點**（另外的電腦）| — | — | 100 GB+ | NVIDIA + ≥12 GB VRAM |

### 軟體需求

| 軟體 | 版本 | 為什麼需要 |
|------|------|---------|
| Docker Engine | 24.x+ | 容器執行 |
| Docker Compose | v2.x+ | 服務編排 |
| Python | 3.9+ | 跑 `setup_env.py` 生成 secrets |
| Git | 任意 | 取得程式碼（或用 zip 下載） |

> 💡 **不需要**：Python venv、Node.js、PostgreSQL、Redis 等等。**全部跑在 Docker 內**。

## 2. 安裝 Docker

### Windows 11

1. 下載 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
2. 安裝時**勾選「Use WSL 2 instead of Hyper-V」**
3. 安裝完成 → 重開機 → 啟動 Docker Desktop → 等右下角圖示變綠
4. 開啟 PowerShell 或 CMD，驗證：
   ```powershell
   docker version
   docker compose version
   ```

### Ubuntu 22.04 / 24.04

```bash
# 安裝 Docker Engine + compose plugin
sudo apt update
sudo apt install -y docker.io docker-compose-v2

# 把當前使用者加進 docker group（免 sudo）
sudo usermod -aG docker $USER
newgrp docker

# 驗證
docker version
docker compose version
```

### macOS

下載 [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/) → 拖到 Applications → 啟動 → 驗證 `docker version`

## 4. 產生 `.env` 設定檔

專案已提供互動式腳本，會幫你產生**密碼學安全的隨機 secrets**。

### Windows（PowerShell）

```powershell
python scripts\setup_env.py
```

### Linux / macOS

```bash
python3 scripts/setup_env.py
```

### 互動式提示會問你

| 提示 | 建議答案 |
|------|---------|
| 服務層 IP | 區網內固定 IP（例 `192.168.1.50`），開發時可填 `127.0.0.1` |
| 資料庫路徑 | 預設 `/data/ai_platform.db` 即可 |
| Token 月度配額 | 預設 `5000000` 即可（每位使用者 5M tokens / 月） |
| SMTP 信箱設定 | 學生環境可暫時跳過（直接 Enter 不填） |
| Storage Mount Path（給 GPU Worker） | 服務層這台可暫時不填 |

### 腳本會做的事

- ✅ 用 `secrets.token_urlsafe()` 產生：
  - `JWT_SECRET_KEY`（64 字元）
  - `WORKER_API_TOKEN`（32 字元）
  - `WEBUI_SECRET_KEY`（32 字元）
  - `SECRETS_MASTER_KEY`（64 字元，v2.0 Lab 用）
- ✅ 寫入 `./.env`（服務層）
- ✅ 寫入 `./gpu-worker/.env`（GPU 節點用，token 自動對齊）
- ✅ 備份既有 `.env` 為 `.env.backup-YYYY-MM-DD`
- ✅ 印出設定摘要

### 完成後驗證

```bash
ls -la .env gpu-worker/.env
```

兩個檔案都應該存在。**請把它們列為機密**（不要 commit 進 git）。

## 7. 建立第一個管理員帳號 (Bootstrap)

> ⚠️ **為什麼這步特別**：基於安全考量，我們**移除了硬編碼的 `admin/admin` 預設帳號**。
> 一般使用者註冊 (`POST /register`) 只能建立 `student` 角色，無法自行提升為 admin。
> 因此「第一個 admin」必須由你直接寫入 DB（雞生蛋問題）。

### 方法 A：用 Python 腳本直接寫 DB（推薦）

在 CodeSpace 根目錄下執行：

```bash
docker compose exec job-scheduler python -c "
from app.database import SessionLocal
from app.crud import get_password_hash
from app import models
from datetime import datetime, timezone, timedelta

db = SessionLocal()

# === 修改這 3 個欄位 ===
ADMIN_USERNAME = 'admin'
ADMIN_EMAIL    = 'admin@school.edu.tw'
ADMIN_PASSWORD = 'CHANGE_THIS_STRONG_PASSWORD'   # 至少 12 字元
# =======================

# 檢查是否已存在
existing = db.query(models.User).filter(models.User.username == ADMIN_USERNAME).first()
if existing:
    print(f'User {ADMIN_USERNAME} already exists, skipping.')
else:
    user = models.User(
        username=ADMIN_USERNAME,
        email=ADMIN_EMAIL,
        hashed_password=get_password_hash(ADMIN_PASSWORD),
        role='admin',
        is_active=1,
    )
    db.add(user)
    db.flush()

    # 同步建立 token quota
    quota = models.TokenUsage(
        user_id=user.id,
        tokens_used=0,
        tokens_limit=99_000_000,
        reset_date=datetime.now(timezone.utc) + timedelta(days=30),
    )
    db.add(quota)
    db.commit()
    print(f'Admin user created: id={user.id} username={ADMIN_USERNAME}')
db.close()
"
```

執行成功會印出：

```
Admin user created: id=<uuid> username=admin
```

### 方法 B：互動式 shell（適合不熟 Python 的人）

```bash
docker compose exec job-scheduler python
```

進入 REPL 後，把 7-A 的腳本貼上去就行。

### 立即更換密碼

用瀏覽器：

1. 開啟 http://localhost:8888/

或者直接執行 7-A 但用一個你會記得的「臨時密碼」，登入後在 UI 改成最終密碼。

## 8. 建立一般使用者 / 教師帳號

### 方法 A：管理員透過 UI 配發（推薦，會自動寄信通知）

1. 用 admin 帳號登入 http://localhost:8888/
2. 進入「**使用者管理**」分頁
3. 點「**Provision User**」按鈕
4. 填寫：
   - Username（學號或姓名拼音）
   - Email（會收到臨時密碼通知信）
   - Role：`student` / `teacher` / `admin`（可一次直接給 teacher 或更多 admin）
   - Password：可留空（自動產生）或自訂

點建立後，系統會：
- 建立帳號 + Token 額度
- 寄出臨時密碼通知信（前提是 SMTP 已設定）
- 使用者登入後可自行改密碼

### 方法 B：使用者自助註冊（僅限 student）

1. 瀏覽器打開 http://localhost/train/
2. 點「**註冊**」按鈕
3. 填寫 username / email / password → 建立
4. 直接登入

> ⚠️ 自助註冊**強制 role=student**（schema 層面已擋下，無法手動改 admin / teacher）。

### 方法 C：批次匯入（多人時用）

如果需要一次匯入 30 個學生，用 Python 腳本：

```bash
docker compose exec job-scheduler python << 'EOF'
from app.database import SessionLocal
from app.crud import get_password_hash
from app import models
from datetime import datetime, timezone, timedelta

# === 修改這個清單 ===
USERS = [
    ('T1090001', 'T1090001@school.edu.tw', 'T1090001'),
    ('T1090002', 'T1090002@school.edu.tw', 'T1090002'),
    # ...
]
# ====================

db = SessionLocal()
created = 0
for username, email, password in USERS:
    if db.query(models.User).filter(models.User.username == username).first():
        continue
    user = models.User(
        username=username, email=email,
        hashed_password=get_password_hash(password),
        role='student', is_active=1,
    )
    db.add(user)
    db.flush()
    db.add(models.TokenUsage(
        user_id=user.id, tokens_used=0, tokens_limit=5_000_000,
        reset_date=datetime.now(timezone.utc) + timedelta(days=30),
    ))
    created += 1
db.commit()
print(f'Created {created} users')
db.close()
EOF
```

### 方法 D：Mock SSO（教學用，最快）

`job-scheduler/app/sso_policy.yaml` 內預設了 2 個學生：
- `T1090001` / 密碼 `T1090001`
- `T1090002` / 密碼 `T1090002`

可直接到登入頁點「SSO 登入」用這兩個試。**不建議用在正式環境**。

## 9. （選用）建立 v2.0 Lab base images

> 如果你要使用 **v2.0 Lab**（VS Code in Browser + 7 個學期鎖定 image），需要先 build 這些 image。
> 第一次 build 約需 **30–60 分鐘**（含模型預下載），但只需要做一次。
>
> 不打算用 Lab 也可以跳過本節。

### 9.1 Build 7 個 base images

```bash
./infrastructure/base-images/build-all.sh
```

build 完用 `docker images | grep aibase` 確認有 7 個。

### 9.2 預下載熱門模型到共享快取

```bash
./infrastructure/base-images/preload-models.sh
```

會把 Llama-2-7B / Mistral-7B / BERT / T5 等模型下載到 `shared_models` volume，所有使用者共用。
**佔約 200 GB 磁碟**，請確認空間足夠。

### 9.3 重啟服務

```bash
docker compose down
docker compose up -d
```

### 9.4 驗證 Lab 可用

登入 → 運算任務 → Notebook → 選擇 image → 點「開啟 Notebook」→ 5–10 秒後跳到 `/code/{user_id}/`，看到 VS Code 介面就成功了。

## 10. 常見問題排查

### Q1：scheduler 容器不是 healthy

```bash
docker compose logs job-scheduler | tail -30
```

**常見錯誤**：

| 錯誤訊息 | 原因 | 修法 |
|---------|------|------|
| `JWT_SECRET_KEY too short` | `.env` 沒設或值太短 | 重跑 `python scripts/setup_env.py` |
| `JWT_SECRET_KEY uses an insecure default value` | `.env` 用了黑名單預設值 | 同上 |
| `WORKER_API_TOKEN must be set in .env` | `.env` 完全沒這個變數 | 同上 |
| `SECRETS_MASTER_KEY too short` | v2.0 新增的變數沒設 | 同上 |
| `no such file: /data/ai_platform.db` | data 目錄權限問題 | `mkdir -p data && chmod 777 data` |

### Q2：可以連 http://localhost/health 但看不到網頁

確認 nginx 容器啟動：
```bash
docker compose ps nginx
```

如果 nginx 不在 running，看 log：
```bash
docker compose logs nginx
```

### Q3：忘了 admin 密碼

回到第 7 節，用方法 A 但加入「強制更新」邏輯：

```bash
docker compose exec job-scheduler python -c "
from app.database import SessionLocal
from app.crud import get_password_hash
from app import models

db = SessionLocal()
admin = db.query(models.User).filter(models.User.username == 'admin').first()
if admin:
    admin.hashed_password = get_password_hash('NEW_PASSWORD_HERE')
    db.commit()
    print('Password reset')
else:
    print('Admin user not found')
db.close()
"
```

### Q4：要重新開始（清空所有資料）

```bash
docker compose down -v   # 注意 -v 會刪除 volume
rm -rf data/ai_platform.db
docker compose up -d --build
```

⚠️ 這會**永久刪除所有使用者、任務、聊天紀錄**。確認後再用。

### Q5：要看完整服務 log

```bash
# 即時 log
docker compose logs -f job-scheduler

# 最後 100 行
docker compose logs --tail 100 job-scheduler
```

### Q6：服務跑得起來但不接受我的登入

- 確認你打的 username / password 大小寫正確（username 區分大小寫）
- 確認帳號的 `is_active = 1`
- 直接查 DB：
  ```bash
  docker compose exec job-scheduler python -c "
  from app.database import SessionLocal
  from app import models
  db = SessionLocal()
  for u in db.query(models.User).all():
      print(u.username, u.role, u.is_active)
  db.close()
  "
  ```

### Q7：要設定 SMTP（讓系統可以寄密碼通知信）

編輯 `.env`，取消註解這幾行並填入：

```
SMTP_SERVER=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your-email@gmail.com
SMTP_PASSWORD=your-app-password    # Gmail 要用 App Password，不是登入密碼
SMTP_FROM_EMAIL=your-email@gmail.com
```

然後 `docker compose restart job-scheduler` 即可。
