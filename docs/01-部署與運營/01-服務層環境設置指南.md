# 01 - 環境設置指南 | Environment Setup Guide

## 系統架構配置 | System Architecture

本平台採用**主從式分離架構 (Client-Server Architecture)**：

### 1. GPU高階伺服器 (GPU High-End Server)
- **作業系統**：Windows 11 (或特定 Linux 發行版)
- **角色**：負責實際執行 AI 模型訓練與高耗能運算的資源節點。

### 2. 服務層 (Service Layer)
- **作業系統**：Ubuntu 22.04 / 24.04 LTS
- **角色**：負責承載所有的 Docker 容器、AI 模型網關與後端排程器。系統維護指令主要於此層執行。
- **必備軟體**：
  - Docker Engine (版本 24.x+)
  - Docker Compose (版本 2.x+)

### 3. 工作站 (Workstation)
- **角色**：僅作為終端存取設備，**不需安裝**任何開發環境或 Docker 服務。
- **使用方式**：終端使用者與管理員僅需透過瀏覽器，連線至服務層提供的網頁 (例如：`http://<服務層IP>/train/`)，即可進行所有操作與管理。

---

## 🌍 跨作業系統相容性與前置條件 | Cross-OS Compatibility & Prerequisites

本專案全面採用 **Docker 容器化**，不再依賴特定的作業系統環境腳本（如 `.bat`, `.sh`, `.ps1`）。只要伺服器能運行 Docker，即可在 Windows、Linux (Ubuntu/CentOS 等) 或 macOS 上無縫部署。

唯一的差異在於「安裝 Docker」以及「啟動指令」：

| 作業系統 | Docker 安裝方式 | 啟動指令 | 檔案路徑格式 |
|---------|---------------|---------|------------|
| **Windows** | 安裝 [Docker Desktop](https://www.docker.com/products/docker-desktop/) (需啟用 WSL2) | `docker-compose up -d --build` | `\` 或 `/` 皆可 |
| **Linux (Ubuntu)** | `sudo apt install docker.io docker-compose-v2` | `docker compose up -d --build` | 僅支援 `/` |
| **macOS** | 安裝 [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/) | `docker compose up -d --build` | 僅支援 `/` |

> [!NOTE]
> 專案資料夾內遺留的 `scripts/deploy.sh` 僅是為了 Linux/macOS 開發者方便使用的連續技腳本（將上述步驟整合為一鍵），**並非系統運作的必要依賴**。Windows 使用者直接執行 docker 指令即可。

## 快速管理指令 | Control Commands

### 🚀 全域控制 (啟動/關閉全部功能)
當您需要完整的 AI 訓練助理環境時，請使用組合指令。這對於確保模型釋放 GPU 資源非常重要。

```bash
# 一鍵啟動 (包含所有核心與模型)
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml up -d --build

# 一鍵關閉 (徹底清理所有服務與網路)
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml down
```

### 🛠️ 分層維運 (推薦開發使用)
如果您只是修改 UI 或業務邏輯，不需要頻繁重啟龐大的 AI 模型：

```bash
# 僅啟動/更新核心服務 (省去檢查模型容器)
docker-compose up -d

# 僅重啟後端排程器
docker-compose restart job-scheduler
```

### Step 4: 存取服務介面 | Access Services

本平台已完成介面整合，**TRAIN_HUD** 為所有功能的單一入口：

| 服務項目 Service | 入口路徑 URL | 功能說明 Description |
|-----------------|-------------|-------------------|
| **TRAIN_HUD** | [`http://localhost/train/`](http://localhost/train/) | **主要入口**：整合 AI 大廳 (AI Hub)、對話助理與任務管理。 |
| **管理員面板** | [`http://localhost:8888/`](http://localhost:8888/) | **Admin UI**：獨立於 Port 8888，包含系統設定檔管理、叢集監控等。 |
| **AI Hub (四宮格)** | 於 Assistant 分頁內 | **功能導覽**：提供模型對話、文書、影音等分類入口。 |
| **輔助對話介面** | `http://localhost/` | **Open WebUI**：備用對話面板。 |
| **API 互動文件** | `http://localhost:8002/docs` | **Swagger UI**：後端介面測試與開發參考。 |
| **健康檢查** | `http://localhost/health` | **Gateway Status**：Nginx 運作狀態。 |

> [!IMPORTANT]
> **連接埠說明 (Ports Information)**：
> - **Port 80**：Nginx API Gateway 統一入口（推薦使用，路徑為 `/train/`）。
> - **Port 8888**：獨立的管理員介面與管理員 API，用於避免從使用者介面登入被暴力破解。
> - **Port 8002**：Job Scheduler 直接入口（僅後端開發測試使用）。
> - **Port 8080**：Open WebUI 直接入口（僅備援使用）。

## 🛠️ 開發與撰寫指引 | Development & Writing Guide

### 1. 修改網頁介面 (Frontend)
如果您要更換視覺風格、修改聊天室佈局：
- **位置**：`web-ui/` (一般使用者), `admin-ui/` (管理員專用)
- **關鍵檔案**：
    - `web-ui/index.html`: 包含運算任務 (Compute Tasks) 左右分欄、AI Assistant 大廳。
    - `admin-ui/index.html`: 獨立的管理員介面，包含資源監控、使用者管理與系統設定檔管理。
    - `web-ui/app.js`: 處理串流對話 (Fetch Stream API)、分頁切換。
    - `admin-ui/admin.js`: 處理管理員權限驗證、API 呼叫與設定檔讀寫。

### 2. 管理 GPU 資源與排程政策 (Resources)
- **位置**：`job-scheduler/app/scheduler_policy.yaml`
- **說明**：在此定義實體 GPU 伺服器的節點池資訊。

### 3. 編寫 AI 助手與後端邏輯 (Backend)
- **位置**：`job-scheduler/app/routers/`
- **檔案**：
    - `chat.py`: **[新]** 負責 LLM 串流代理與紀錄存儲。
    - `jobs.py`: 負責訓練任務的 CRUD 與狀態更新。
    - `admin.py`: **[新]** 管理員專屬端點 (使用者/模型/任務全域查詢)。
    - `auth.py`: 認證與登入 (含 SSO 混合驗證與 IP 追蹤)。

## 服務管理 | Service Management

```bash
# 查看所有服務狀態
docker-compose -f docker-compose.yml -f docker-compose.ai-models.yml ps

# 重啟核心後端
docker-compose restart job-scheduler

# 停止服務 (保持容器)
docker-compose stop

# 關閉並移除容器 (常用)
docker-compose down

# 完全清理 (移除容器、網路與磁碟卷)
# 警告：這會刪除儲存在 Docker 卷中的資料
docker-compose down -v
```

## 🌐 遠端存取與部署 | Remote Access

本平台支援本機部署後供同區網內的其他機器存取。

### 1. 取得主機 IP
在執行 Docker 的服務層主機 (Ubuntu) 上執行：
```bash
hostname -I    # Ubuntu
# 尋找 IPv4 位址 (例如: 192.168.1.105)
```

### 2. 跨機器連結
在另一台電腦的瀏覽器中輸入：
- `http://[主機IP]/train/`

### 3. 防火牆設定 (重要)
若其他機器無法連線，請檢查主機的防火牆設定：
- **開放埠號**：確保 **Port 80** (HTTP 用戶端), **Port 8888** (管理員介面) 與 **Port 8002** (後端 API) 已設為防火牆允許規則 (Inbound Rules)。
- **網路類型**：建議將網路設為「專用 (Private)」。

## 常見問題 | FAQ

### Q: 為什麼 AI 助手沒有回覆？
1. 請確認 `ai-platform-portkey` 容器是否正常運作。
2. 檢查 `.env` 中的 API Key 是否正確設定。
3. 查看日誌：`docker-compose logs -f job-scheduler` 確認串流轉發有無錯誤。

### Q: 如何切換回舊版 Open WebUI？
您可以存取 `http://localhost/` 直接使用 Open WebUI，但該處對話紀錄與 `TRAIN_HUD` 物理隔離。
