# 10 - 正式上線轉換指南 | Production Migration Guide

本文件說明將 AI 訓練平台從「Windows 開發測試環境」遷移至「Ubuntu 正式運行環境」時，所有需要修改的項目與步驟。

---

## 📌 前提：為什麼服務層可以在 Windows 上測試？

服務層的所有核心服務（Nginx、Job Scheduler、Portkey、Ollama）皆封裝於 **Docker 容器**內部。Docker 容器本身是跨平台的——無論宿主機 (Host OS) 是 Windows、macOS 或 Linux，容器內部的環境一律是 Linux。因此：

| 環境 | 宿主機 OS | Docker 內部 | 能否正常運行 |
|------|-----------|-------------|:---:|
| **開發測試** | Windows 11 + Docker Desktop | Linux 容器 | ✅ |
| **正式上線** | Ubuntu 22.04 + Docker Engine | Linux 容器 | ✅ |

> [!IMPORTANT]
> **唯一例外**：`scripts/deploy.sh` 是 Bash 腳本，無法直接在 Windows PowerShell 中執行。開發測試時，請直接使用 `docker compose up -d --build` 替代。

---

## 🔄 上線前必做修改清單 (Checklist)

### 1️⃣ `.env` 環境變數 (最重要)

| 變數名稱 | 開發測試值 (Windows) | 正式上線值 (Ubuntu) | 說明 |
|----------|---------------------|--------------------|----|
| `JWT_SECRET_KEY` | `default-insecure-secret-key` | **隨機 32+ 字元字串** | 安全性核心，必須更換 |
| `WEBUI_SECRET_KEY` | `defaul-secret-key-please-change` | **隨機 32+ 字元字串** | Open WebUI 的加密金鑰 |
| `GPU_MOCK_MODE` | `true` | `false` | 關閉模擬模式 |
| `GPU_SERVER_1_HOST` | `192.168.1.100` | **真實 GPU 伺服器 IP** | 替換為實際 IP |
| `ANTHROPIC_API_KEY` | `sk-ant-placeholder` | **真實 API Key** | 否則 AI 助手無法使用 Anthropic 模型 |
| `OPENAI_API_KEY` | `sk-placeholder` | **真實 API Key** | 否則 AI 助手無法使用 GPT 模型 |
| `GOOGLE_API_KEY` | `placeholder` | **真實 API Key** | 否則 AI 助手無法使用 Gemini 模型 |

> [!CAUTION]
> **JWT_SECRET_KEY 未更換就上線，等同於讓所有人都能偽造管理員 Token。**
> 
> 產生安全金鑰的指令：
> ```bash
> openssl rand -hex 32
> ```

---

### 2️⃣ `scheduler_policy.yaml` 排程設定

檔案路徑：`job-scheduler/app/scheduler_policy.yaml`

| 欄位 | 開發測試值 | 正式上線值 |
|------|-----------|-----------|
| `mock_mode` | `true` | `false` |

---

### 4️⃣ SSO 登入模式切換

檔案路徑：`.env` + `job-scheduler/app/sso_policy.yaml`

| 設定 | 開發測試值 | 正式上線值 |
|------|-----------|-----------|
| `SSO_MOCK_MODE` (.env) | `true` | `false` |
| `cas.server_url` (yaml) | 不使用 | 改為學校真實 SSO URL |
| `cas.service_url` (yaml) | `http://localhost/...` | 改為正式環境外部 IP/域名 |

> [!NOTE]
> 詳細的 SSO 設定步驟請參考 [05-SSO整合設定指南.md](05-SSO整合設定指南.md)。

---

### 5️⃣ Ollama GPU 加速 (選用)

若 Ubuntu 服務層本身配有 NVIDIA GPU 並需要跑本地模型推論，請修改 `docker-compose.ai-models.yml`：

```yaml
ollama:
  image: ollama/ollama:latest
  # ↓ 取消以下兩行的註解
  runtime: nvidia
  environment:
    - NVIDIA_VISIBLE_DEVICES=all
```

> [!NOTE]
> 這與 GPU 高階伺服器 (Windows 11) 的「訓練任務 GPU」是不同的用途。此處的 GPU 用於 Ollama 本地模型推論。

---

### 6️⃣ 防火牆開放 (Ubuntu)

```bash
# 開放 Web 入口 (Port 80)
sudo ufw allow 80/tcp

# 開放 API Docs (僅限開發期間，正式上線建議移除)
# sudo ufw allow 8002/tcp

# 啟用防火牆
sudo ufw enable
```

---

### 7️⃣ deploy.sh 部署腳本

在 Ubuntu 上，您可以直接使用 `scripts/deploy.sh` 一鍵部署：
```bash
chmod +x scripts/deploy.sh
./scripts/deploy.sh
```

> [!TIP]
> Windows 開發測試時無法執行此腳本，請直接使用：
> ```powershell
> docker compose up -d --build
> ```

---

## 📋 上線前最終驗證清單

請逐項確認後再對外開放服務：

- [ ] `.env` 中的 `JWT_SECRET_KEY` 已替換為安全的隨機字串
- [ ] `.env` 中的 `WEBUI_SECRET_KEY` 已替換為安全的隨機字串
- [ ] `.env` 中的 `GPU_MOCK_MODE` 已設為 `false`
- [ ] `scheduler_policy.yaml` 中的 `mock_mode` 已設為 `false`
- [ ] GPU 伺服器上的 `docker-compose.yml` 已設定好正確的主機 IP
- [ ] GPU 伺服器的 Worker 容器已成功啟動並出現 `Polling` 日誌
- [ ] LLM API Keys 已填入真實值（至少一組）
- [ ] SSO 模式已按需求設定（Mock 或 Real CAS）
- [ ] Ubuntu 防火牆已開放 Port 80
- [ ] 使用 `docker compose ps` 確認所有容器為 `healthy` / `running`
- [ ] 從工作站瀏覽器可正常存取 `http://<服務層IP>/train/`

## 🏗️ 環境差異速查表

| 項目 | Windows 開發測試 | Ubuntu 正式上線 |
|------|:---:|:---:|
| Docker 工具 | Docker Desktop | Docker Engine + Compose Plugin |
| 部署方式 | `docker compose up -d --build` | `./scripts/deploy.sh` 或 `docker compose` |
| GPU Mock | ✅ 開啟 | ❌ 關閉 |
| Worker Agent | 未啟動 (主機單機測試) | 需在 GPU 伺服器啟動 `docker-compose up` |
| JWT 金鑰 | 可使用預設值 | **必須替換** |
| SSO | Mock 模式 | Real CAS 或 Mock |
| 防火牆 | 手動開放 Port 80 | `ufw allow 80/tcp` |
| deploy.sh | ❌ 無法執行 | ✅ 可執行 |
| Ollama GPU | CPU-only | 可啟用 NVIDIA runtime |

## 🔑 附錄 A：安全金鑰生成方式

### A-1. 產生 32+ 字元的安全隨機字串

適用於 `JWT_SECRET_KEY` 和 `WEBUI_SECRET_KEY`。以下提供三種方式，擇一即可：

**方法一：openssl（推薦，Ubuntu 內建）**
```bash
openssl rand -hex 32
# 輸出範例：a3f7c9e1b45d28f06a9c3e71d84b2f0e5c7a8d3b6e9f2c4a1d7b0e8f3c6a9d2b
```

**方法二：Python（跨平台皆可用）**
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

**方法三：PowerShell（Windows 開發時使用）**
```powershell
-join ((1..64) | ForEach-Object { '{0:x}' -f (Get-Random -Maximum 16) })
```

> [!IMPORTANT]
> 產生後請**立即**貼入 `.env` 檔案中對應的欄位，並且**不要**將 `.env` 提交到 Git（`.gitignore` 已排除）。

## 🔑 附錄 B：雲端 LLM API Key 取得方式

以下為各家模型供應商的 API Key 申請流程。您只需申請**至少一組**即可讓 AI 助手正常運作。

### B-1. Google Gemini API Key（免費額度最高，推薦優先申請）

| 項目 | 內容 |
|------|------|
| **申請網址** | [https://aistudio.google.com/apikey](https://aistudio.google.com/apikey) |
| **費用** | 免費方案每分鐘 15 次請求，足夠開發與小規模使用 |
| **前置條件** | Google 帳號 |

**步驟**：
1. 前往 [Google AI Studio](https://aistudio.google.com/apikey)，以 Google 帳號登入。
2. 點擊「**Create API key**」→ 選擇一個 Google Cloud 專案（或新建一個）。
3. 複製生成的 Key（格式如 `AIzaSy...`）。
4. 貼入 `.env`：
   ```env
   GOOGLE_API_KEY=AIzaSy您的真實Key
   ```

---

### B-2. OpenAI API Key（GPT-4o / GPT-4 / GPT-3.5）

| 項目 | 內容 |
|------|------|
| **申請網址** | [https://platform.openai.com/api-keys](https://platform.openai.com/api-keys) |
| **費用** | 預付制，新帳號有 $5 免費額度（視地區而定） |
| **前置條件** | OpenAI 帳號 + 信用卡（僅驗證，免費額度內不扣款） |

**步驟**：
1. 前往 [OpenAI Platform](https://platform.openai.com/api-keys)，登入或註冊帳號。
2. 點擊「**Create new secret key**」，輸入名稱後按建立。
3. 複製金鑰（格式如 `sk-proj-...`）。⚠️ 此金鑰只會顯示一次。
4. 貼入 `.env`：
   ```env
   OPENAI_API_KEY=sk-proj-您的真實Key
   ```

---

### B-3. Anthropic API Key（Claude 3.5 / Claude 3）

| 項目 | 內容 |
|------|------|
| **申請網址** | [https://console.anthropic.com/settings/keys](https://console.anthropic.com/settings/keys) |
| **費用** | 預付制，新帳號有 $5 免費額度 |
| **前置條件** | Anthropic 帳號 + 信用卡 |

**步驟**：
1. 前往 [Anthropic Console](https://console.anthropic.com/)，註冊或登入。
2. 進入「**Settings → API Keys**」頁面。
3. 點擊「**Create Key**」，輸入名稱後按建立。
4. 複製金鑰（格式如 `sk-ant-api03-...`）。
5. 貼入 `.env`：
   ```env
   ANTHROPIC_API_KEY=sk-ant-api03-您的真實Key
   ```

---

### B-4. 一次設定範例

以下為一個完成所有金鑰設定後的 `.env` 片段範例：

```env
# ===== 安全金鑰 (用 openssl rand -hex 32 產生) =====
JWT_SECRET_KEY=a3f7c9e1b45d28f06a9c3e71d84b2f0e5c7a8d3b6e9f2c4a1d7b0e8f3c6a9d2b
WEBUI_SECRET_KEY=7d2a8f4c1e6b3a9d0f5c2e8b4a7d1c6f3e9b5a2d8c4f0e7b3a6d9c1f5e2b8a4d

# ===== 雲端 LLM API Keys =====
GOOGLE_API_KEY=AIzaSyBxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
ANTHROPIC_API_KEY=sk-ant-api03-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# ===== GPU 設定 =====
GPU_MOCK_MODE=false
```

> [!TIP]
> **最低成本方案**：如果預算有限，建議只申請 **Google Gemini API Key**（免費額度最高），即可讓 AI 助手功能完整運作。其餘 Key 可留為 `placeholder`，系統會在使用者選擇該模型時回報錯誤，但不影響其他模型的運作。
